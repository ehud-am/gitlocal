# GitLocal v0.7.0 Release Review

## Scope

- Release: `v0.7.0`
- Feature: file, folder, and repository handles with unified open behavior; regular-folder capability parity; deferred git context loading; plain-folder badge cleanup
- Branch: `014-folder-repo-capabilities`
- Release date: 2026-05-18

## Verification Reviewed

- `npm run verify` passed locally with server tests, UI tests, production build, and dependency audits.
- Server test suite passed: 9 files, 239 tests.
- UI test suite passed: 18 files, 196 tests.
- Root npm audit reported 0 vulnerabilities.
- UI npm audit reported 0 vulnerabilities.
- `npm pack --dry-run` passed for `gitlocal@0.7.0`; the package contains 14 files and the built server/UI bundles.
- `npm view gitlocal version` returned `0.6.3`, confirming `0.7.0` is the next unpublished package version.

## Contrarian QA Findings

Contrarian QA sub-agent review ran before the release commit and identified release-readiness issues:

- File handles were documented but still rejected by the runtime open endpoint.
  - Result: Fixed. The open handler now accepts files, resolves the correct root, and persists the selected file in viewer state.
- Git repository startup still duplicated root tree loading and performed per-entry ignore checks.
  - Result: Fixed. The content panel shares the root tree query key with the file tree, and ignored status is batched through one `git check-ignore --stdin` call per directory listing.
- The release review artifact was missing.
  - Result: Fixed by this release review.
- Plain folder picker rows still displayed a `local` tag.
  - Result: Fixed. Picker entries only show a `git` tag for repositories; regular files and folders have no local-only badge.
- Git failure tests could still inherit ambient identity/configuration from the developer machine.
  - Result: Fixed. Test git commands now run with isolated git identity and config environment values.
- `git diff --check` reported trailing whitespace in the release spec plan.
  - Result: Fixed and rechecked.
- The npm package description still described only repository browsing.
  - Result: Fixed. The package now describes browsing local folders and git repositories.

No remaining functional blocker was found in the picker open flow, regular-folder badge behavior, deferred git context model, or repository startup performance changes.

## Accessibility Review

- The unified picker keeps a single primary Open action and does not introduce additional keyboard stops for obsolete folder/repository split actions.
- Removing plain-folder `local` badges also removes redundant visual noise from regular-folder rows.
